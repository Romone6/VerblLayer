# Support

## Before the public repository exists

Community support is not available yet. Do not use source-code issues as a substitute for a private security channel.

## After publication

Use GitHub Discussions for installation questions, deployment notes, and workflow design. Use issues only for reproducible defects or narrowly scoped feature proposals. Use the private contact in `SECURITY.md` for vulnerabilities.

## What to include

Include the VerblLayer version or commit, deployment mode, redacted logs, the exact command run, and the expected versus actual result. Never include API keys, database URLs, proxy secrets, or customer data.
